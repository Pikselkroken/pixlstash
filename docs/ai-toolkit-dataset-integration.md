# PixlStash → AI-Toolkit Dataset Integration Guide

This document gives an AI-coding agent enough context to build an ai-toolkit
dataset loader (or config extension) that reads images and captions from a
running PixlStash instance.

---

## 1. Authentication

PixlStash uses **session cookies**. Every request must carry a `session_id`
cookie obtained by logging in. The recommended flow for a script/tool is:

1. **Create a personal API token** in PixlStash (Settings → API Tokens).
2. **Exchange the token for a session cookie** with a single login call.
3. **Re-use the cookie** for all subsequent requests.

```python
import requests

BACKEND = "http://localhost:9537"   # change to your server address
TOKEN   = "paste-your-api-token-here"

session = requests.Session()

# Exchange the API token for a session cookie
r = session.post(f"{BACKEND}/login", json={"token": TOKEN})
r.raise_for_status()
# All subsequent calls on `session` automatically carry the cookie.
```

> **Note:** The default port is `9537`. On HTTPS deployments you may need to
> set `verify=False` or point to the right cert bundle if you are using a
> self-signed certificate.

## 2. Discover Characters

```
GET /characters
```

Returns a list of all characters.

**Response shape (array):**
```json
[
  {
    "id": 1,
    "name": "Alice",
    "description": "short bio / training trigger text",
    "project_id": 2
  }
]
```

Filter by name:
```
GET /characters?name=Alice
```

Get a single character:
```
GET /characters/{id}
```

---

## 3. Discover Picture Sets

```
GET /picture-sets
```

Returns a list of all picture sets.

**Response shape (array):**
```json
[
  {
    "id": 7,
    "name": "Training set – Alice portraits",
    "description": "...",
    "project_id": 2
  }
]
```

Get the picture IDs that belong to a set:
```
GET /picture-sets/{id}/members
```

**Response shape (array):**
```json
[
  { "id": 101, "score": 95, ... },
  { "id": 102, "score": 88, ... }
]
```

Each object contains the full picture record (same shape as `GET /pictures/{id}/metadata`).

---

## 4. Get All Picture IDs for a Character

Characters do not have a direct "list pictures" endpoint; pictures are linked
through face assignments. Use the metadata search endpoint filtered by character:

```
GET /pictures/list?character_id={id}
```

This returns picture records in the same shape as the metadata endpoint (see §5).

Alternatively, the picture search endpoint supports character filtering:
```
GET /pictures/search?character_id={id}
```

## 5. Picture Metadata and Captions

```
GET /pictures/{id}/metadata
```

Returns metadata + tags for a single picture.

**Key fields:**
| Field | Type | Description |
|---|---|---|
| `id` | int | Unique picture id |
| `file_path` | str | Relative path inside the vault (not web-accessible directly) |
| `description` | str\|null | Florence-2 natural-language caption generated by PixlStash |
| `tags` | `[{id, tag}]` | WD14 tagger output as an array of tag objects |
| `width` | int | Pixels |
| `height` | int | Pixels |
| `format` | str | e.g. `"jpg"`, `"png"`, `"webp"` |
| `score` | int\|null | User quality score (−1 to 100) |
| `aesthetic_score` | float\|null | Aesthetic quality metric |
| `original_file_name` | str\|null | Filename as uploaded |
| `comfyui_positive_prompt` | str\|null | Positive prompt extracted from embedded ComfyUI workflow metadata |

---

### 5a. Natural-language descriptions (`description`)

PixlStash runs **Florence-2** on every imported image to produce a free-text
English caption stored in the `description` field.

```
"A portrait of a young woman with long red hair, wearing a blue dress, smiling."
```

- Best for models that benefit from sentence-style captions (e.g. FLUX/SD3 text
  encoders that read natural language well).
- May be `null` if Florence-2 has not run yet, or if the model failed to load on
  that server.

**Extracting just the description:**
```python
description = meta.get("description") or ""
```

---

### 5b. Tag-based captions (`tags`)

PixlStash also runs the **WD14** anime/illustration tagger on every image,
producing a list of short descriptor tokens stored in the `tags` array.

```json
[
  {"id": 12, "tag": "1girl"},
  {"id": 47, "tag": "long_hair"},
  {"id": 88, "tag": "blue_dress"}
]
```

- Best for models trained with Danbooru-style tag captions (most SDXL/SD1.5
  LoRA workflows).
- Tags are normalised (underscores, lowercase). The empty-string sentinel tag
  `""` is used internally to signal "tagged but no tags found" — filter it out.
- Not all images will have tags if the tagger has not run yet.

**Extracting a comma-separated tag string:**
```python
EMPTY_SENTINEL = ""

def tags_to_string(meta: dict) -> str:
    return ", ".join(
        t["tag"] for t in meta.get("tags") or []
        if t.get("tag") and t["tag"] != EMPTY_SENTINEL
    )
```

---

### 5c. Choosing between them

| Situation | Recommended choice |
|---|---|
| Training a concept / style LoRA for FLUX or SD3 | `description` (natural language) |
| Training a character LoRA for SDXL / SD1.5 | `tags` (Danbooru-style) |
| Mixed workflow or fine-tuning both | Use `description` when present, fall back to `tags` |
| The tagger hasn't run but the user supplied a trigger word | Prepend the trigger to whichever caption is available |

**Combined helper that lets the caller choose:**
```python
EMPTY_SENTINEL = ""

def build_caption(
    meta: dict,
    mode: str = "description",   # "description" | "tags" | "both"
    trigger: str = "",
) -> str:
    parts = []
    if trigger:
        parts.append(trigger)

    if mode in ("description", "both"):
        desc = meta.get("description") or ""
        if desc:
            parts.append(desc)

    if mode in ("tags", "both"):
        tag_str = ", ".join(
            t["tag"] for t in meta.get("tags") or []
            if t.get("tag") and t["tag"] != EMPTY_SENTINEL
        )
        if tag_str:
            parts.append(tag_str)

    # Final fallback: use whichever is available
    if len(parts) <= (1 if trigger else 0):
        fallback = meta.get("description") or ", ".join(
            t["tag"] for t in meta.get("tags") or []
            if t.get("tag") and t["tag"] != EMPTY_SENTINEL
        )
        if fallback:
            parts.append(fallback)

    return ", ".join(parts)
```

## 6. Downloading the Image File

```
GET /pictures/{id}/image.{ext}
```

Where `ext` matches the picture's `format` field (e.g. `jpg`, `png`, `webp`).
The server returns the raw image bytes with the correct MIME type.

```python
def download_image(session, backend, pic_id, fmt="jpg") -> bytes:
    r = session.get(f"{backend}/pictures/{pic_id}/image.{fmt}")
    r.raise_for_status()
    return r.content
```

> A convenience shorthand that works regardless of format:
> `GET /pictures/{id}/image.jpg` will always return the image (the
> server ignores the extension and serves whatever format is stored).

## 7. Complete Dataset Construction Pattern

```python
import io
import requests
from PIL import Image

BACKEND = "http://localhost:9537"
TOKEN   = "paste-your-api-token-here"

session = requests.Session()
session.post(f"{BACKEND}/login", json={"token": TOKEN}).raise_for_status()


def iter_character_dataset(character_id: int):
    """Yield (PIL.Image, caption_str) tuples for one character."""
    character = session.get(f"{BACKEND}/characters/{character_id}").json()
    char_name = character["name"]

    pictures = session.get(
        f"{BACKEND}/pictures/list", params={"character_id": character_id}
    ).json()

    for pic in pictures:
        meta = session.get(f"{BACKEND}/pictures/{pic['id']}/metadata").json()
        caption = build_caption(meta, char_name)

        img_bytes = download_image(session, BACKEND, pic["id"], meta.get("format", "jpg"))
        image = Image.open(io.BytesIO(img_bytes)).convert("RGB")
        yield image, caption


def iter_picture_set_dataset(set_id: int):
    """Yield (PIL.Image, caption_str) tuples for one picture set."""
    members = session.get(f"{BACKEND}/picture-sets/{set_id}/members").json()

    for pic in members:
        meta = session.get(f"{BACKEND}/pictures/{pic['id']}/metadata").json()
        caption = build_caption(meta)

        img_bytes = download_image(session, BACKEND, pic["id"], meta.get("format", "jpg"))
        image = Image.open(io.BytesIO(img_bytes)).convert("RGB")
        yield image, caption
```

## 8. Configuration Inputs Needed from the User

An AI-toolkit config block or dataset class needs:

| Input | Required | Description |
|---|---|---|
| `pixlstash_url` | ✅ Yes | The URL of the PixlStash server the user wants to connect to (e.g. `http://192.168.1.10:9537`). The extension must expose this as a user-configurable field so users can point it at their instance. |
| `pixlstash_token` | ✅ Yes | Personal API token (create in PixlStash → Settings → API Tokens) |
| `source_type` | ✅ Yes | `"character"` or `"picture_set"` |
| `source_id` | ✅ Yes | Integer id of the character or picture set |
| `caption_mode` | ✅ Yes | `"description"` (Florence-2 natural language), `"tags"` (WD14 comma list), or `"both"` |
| `trigger_word` | No | Optional prefix prepended to every caption (e.g. the LoRA trigger token) |

## 9. Interactive API Documentation

A running PixlStash server exposes:
- **ReDoc** (read-friendly): `http://<host>:<port>/redoc`
- **Swagger UI** (interactive): `http://<host>:<port>/docs`
- **OpenAPI JSON**: `http://<host>:<port>/openapi.json`
